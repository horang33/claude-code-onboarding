### 졸업 과제: 나만의 Skill 만들기

실제 업무에서 **반복하는 작업 하나**를 골라 Skill로 만들어보세요.

좋은 Skill의 조건:
1. **반복성** — 주 1회 이상 하는 작업
2. **구조화** — 매번 비슷한 패턴으로 진행
3. **자동화 가치** — 수동으로 하면 15분 이상 걸리는 작업

## EXECUTE

### 디자이너 트랙

#### 추천 Skill 아이디어

1. **디자인 리뷰 체크리스트**
   - 입력: Figma 파일 URL
   - 처리: 정합성 체크 (컬러, 타이포, 간격, 컴포넌트 사용)
   - 출력: 체크리스트 + 개선 제안

2. **디자인 핸드오프 문서 생성**
   - 입력: 화면 설명
   - 처리: 상태별 동작 정리 (default, hover, error, loading)
   - 출력: 개발자용 스펙 문서 (마크다운)

3. **경쟁사 디자인 분석**
   - 입력: 분석 대상 서비스명
   - 처리: UI 패턴, 인터랙션, 정보 구조 분석
   - 출력: 구조화된 분석 리포트

#### 실습

아래 예시를 참고해서, Claude Code에게 Skill 생성을 요청하세요:

```
claude-practice 폴더에 디자인 리뷰 Skill을 만들어줘.

위치: .claude/skills/design-review/SKILL.md

기능:
- /design-review 실행 시 Figma 파일 URL을 물어봄
- 컬러, 타이포, 간격, 컴포넌트 사용 정합성 체크
- 체크리스트 형태로 결과 출력
- 마지막에 개선 제안 3가지
```

> 위 예시는 참고용입니다. **본인의 실제 반복 업무**로 바꿔서 만들어보세요!

### 리서처 트랙

#### 추천 Skill 아이디어

1. **인터뷰 분석 리포트**
   - 입력: 녹취록 파일 경로
   - 처리: 핵심 발견, 인용문, 감정 흐름, 숨겨진 니즈 추출
   - 출력: 구조화된 분석 리포트

2. **설문 결과 요약**
   - 입력: 설문 데이터 (CSV/Excel)
   - 처리: 주요 통계, 주관식 답변 테마 분류
   - 출력: 요약 리포트 + 추가 리서치 질문

3. **리서치 위클리 다이제스트**
   - 입력: 이번 주 리서치 활동 메모
   - 처리: 활동 정리, 핵심 인사이트, 다음 주 계획
   - 출력: 팀 공유용 위클리 리포트

#### 실습

아래 예시를 참고해서, Claude Code에게 Skill 생성을 요청하세요:

```
claude-practice 폴더에 인터뷰 분석 Skill을 만들어줘.

위치: .claude/skills/interview-analysis/SKILL.md

기능:
- /interview-analysis 실행 시 녹취록 파일 경로를 물어봄
- 핵심 발견 5개 (인용문 포함)
- 감정 변화 흐름 정리
- 숨겨진 니즈 도출
- 다음 리서치 질문 제안 3가지
```

> 위 예시는 참고용입니다. **본인의 실제 반복 업무**로 바꿔서 만들어보세요!

### PM 트랙

#### 추천 Skill 아이디어

1. **PRD 작성**
   - 입력: 기능 설명 또는 회의 노트
   - 처리: 유저 스토리, 성공 지표, 엣지 케이스 구조화
   - 출력: PRD 초안 (마크다운)

2. **스프린트 회고 정리**
   - 입력: 이번 스프린트 메모/회의록
   - 처리: 완료/미완료 정리, 블로커 분석, 액션 아이템 도출
   - 출력: 회고 리포트 + 다음 스프린트 제안

3. **릴리즈 노트 생성**
   - 입력: 이번 배포 변경 사항
   - 처리: 사용자 관점으로 재구성, 카테고리 분류
   - 출력: 내부용 + 외부용 릴리즈 노트

#### 실습

아래 예시를 참고해서, Claude Code에게 Skill 생성을 요청하세요:

```
claude-practice 폴더에 PRD 작성 Skill을 만들어줘.

위치: .claude/skills/prd-writer/SKILL.md

기능:
- /prd-writer 실행 시 기능 설명을 물어봄
- 기능 개요 작성
- 유저 스토리 3개 도출
- 성공 지표 (KPI) 제안
- 엣지 케이스 3가지 정리
```

> 위 예시는 참고용입니다. **본인의 실제 반복 업무**로 바꿔서 만들어보세요!
