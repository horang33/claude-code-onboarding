# Block 5: Subagent - 위임과 병렬 처리 (~10분)

> 하나의 큰 작업을 여러 "주니어"에게 나눠서 동시에 처리하는 방법.
> 디자이너: 주니어 디자이너에게 위임 / 리서처: 리서치 어시스턴트에게 위임 / PM: 팀원에게 업무 위임

---

## EXPLAIN

> **Before**: 인터뷰 5건을 순서대로 분석 → 5시간
> **After**: 서브에이전트 5개를 병렬로 투입 → 1시간

### Subagent란?

Claude Code에서 `Task` 도구를 사용하면 **별도의 에이전트(서브에이전트)**를 띄워서
작업을 위임할 수 있습니다.

#### 디자이너 트랙

**주니어 디자이너에게 위임**하는 것과 같습니다.
- 시니어 디자이너(나): "이 3개 화면 각각 접근성 체크해줘"
- 주니어 디자이너 3명(서브에이전트): 각자 1개씩 동시에 분석
- 결과: 3배 빠르게 완료

#### 리서처 트랙

**리서치 어시스턴트에게 위임**하는 것과 같습니다.
- 리드 리서처(나): "이 인터뷰 5건 각각 1차 코딩해줘"
- 어시스턴트 5명(서브에이전트): 각자 1건씩 동시에 분석
- 결과: 5배 빠르게 완료

#### PM 트랙

**팀원에게 업무를 위임**하는 것과 같습니다.
- PM(나): "이 3개 경쟁사 각각 기능 비교표 만들어줘"
- 팀원 3명(서브에이전트): 각자 1개씩 동시에 분석
- 결과: 3배 빠르게 비교 완료

### 언제 쓰나?

| 상황 | 서브에이전트 필요? |
|------|-------------------|
| 파일 1개 분석 | 필요 없음 (직접 처리) |
| 파일 5개 각각 분석 | 필요! (병렬 처리) |
| 단순 질문 | 필요 없음 |
| 여러 소스에서 정보 수집 | 필요! (동시 검색) |
| 한 작업의 결과를 다음에 사용 | 필요 없음 (순차 처리) |

### 핵심 규칙

1. **독립적인 작업만 병렬로** — A의 결과가 B에 필요하면 순차 처리
2. **결과는 메인에서 종합** — 서브에이전트는 개별 분석, 종합은 내가
3. **작업 지시는 명확하게** — 서브에이전트에게 "분석해줘"가 아니라 "핵심 발견 3개와 인용문을 추출해줘"

## EXECUTE

### 디자이너 트랙

#### 실습: 디자인 원칙 병렬 분석

Claude Code에서 아래를 입력하세요:

```
아래 3가지 디자인 원칙에 대해 각각 서브에이전트를 띄워서 동시에 분석해줘.
각 서브에이전트는 해당 원칙의 정의, 좋은 예시 1개, 나쁜 예시 1개를 정리해.

1. 시각적 계층구조 (Visual Hierarchy)
2. 일관성 (Consistency)
3. 접근성 (Accessibility)

3개 결과가 모이면 하나의 요약 테이블로 종합해줘.
```

**관찰 포인트**: Claude가 Task 도구를 3번 호출하는 것을 확인하세요.
각 서브에이전트가 독립적으로 작업하고, 마지막에 결과를 종합합니다.

## QUIZ

### 퀴즈 1

```
질문: 서브에이전트를 사용해야 하는 상황은?
옵션:
  A) 모든 작업에 항상 사용
  B) 독립적인 작업 여러 개를 동시에 처리할 때
  C) 어려운 질문을 할 때
정답: B
해설: 서브에이전트는 "독립적인 작업의 병렬 처리"에 적합합니다.
      파일 5개를 각각 분석하거나, 여러 소스에서 동시에 정보를 모을 때 효과적입니다.
      단일 작업이나 순차적 의존성이 있는 작업에는 불필요합니다.
```
