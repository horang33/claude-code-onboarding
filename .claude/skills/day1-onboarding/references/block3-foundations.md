# Block 3: CLAUDE.md + Skill - 영구 기억과 업무 레시피 (~15분)

> 두 가지 핵심 개념을 한 블록에서 배웁니다.
> 디자이너: 브랜드 가이드라인 + 컴포넌트 라이브러리
> 리서처: 리서치 프로토콜 + 분석 프레임워크
> PM: 제품 스펙 문서 + PRD 양식/기능 템플릿

---

## EXPLAIN

### Part 1: CLAUDE.md — AI의 영구 기억

Claude Code와 대화하면 매번 새로운 대화가 시작됩니다.
그런데 매번 "나는 디자이너야, Figma를 써, 한국어로 답해줘"를 반복하면 비효율적이죠.

**CLAUDE.md**는 프로젝트 폴더에 두는 텍스트 파일입니다.
Claude Code가 대화를 시작할 때 **자동으로 읽어서 기억**합니다.

#### 디자이너 트랙

CLAUDE.md는 **브랜드 가이드라인**과 같습니다.
- 브랜드 가이드라인: "우리 브랜드 컬러는 #FF6F61, 톤은 친근하게"
- CLAUDE.md: "나는 Product Designer, Figma를 주로 쓰고, 한국어로 답해줘"

한번 써두면 매번 반복 설명할 필요 없이 Claude가 알아서 맥락을 유지합니다.

#### 리서처 트랙

CLAUDE.md는 **리서치 프로토콜**과 같습니다.
- 리서치 프로토콜: "인터뷰는 반구조화, 녹취 후 어피니티 다이어그램"
- CLAUDE.md: "나는 UX Researcher, 인터뷰 분석 시 핵심 발견+인용문 포맷으로"

한번 써두면 분석 요청마다 같은 포맷으로 결과가 나옵니다.

#### PM 트랙

CLAUDE.md는 **제품 스펙 문서**와 같습니다.
- 제품 스펙 문서: "타겟 유저는 2030, 핵심 지표는 전환율, 우선순위는 P0/P1 구분"
- CLAUDE.md: "나는 PM, PRD 작성 시 유저 스토리+성공 지표 포함, 한국어로 답해줘"

한번 써두면 매번 반복 설명할 필요 없이 Claude가 알아서 맥락을 유지합니다.

#### CLAUDE.md 예시

```markdown
# My Project

## 내 역할
- Product Designer / UX Researcher
- 한국어로 소통

## 선호하는 응답 스타일
- 간결하게 (5개 이하 핵심 포인트)
- 전문 용어는 설명 포함
- 마크다운 포맷 사용

## 자주 쓰는 도구
- Figma (디자인)
- Notion (문서화)
- Slack (커뮤니케이션)
```

### Part 2: Skill — 반복 업무 레시피

같은 유형의 작업을 매번 긴 프롬프트로 요청하면 비효율적입니다.
**Skill**은 자주 하는 작업을 "레시피"로 만들어 한 번에 실행하는 기능입니다.

#### 디자이너 트랙

Skill은 **컴포넌트 라이브러리**와 같습니다.
- 컴포넌트 라이브러리: Button, Card, Modal 등을 미리 만들어두고 재사용
- Skill: "디자인 리뷰", "컴포넌트 정리" 등을 미리 정의해두고 `/명령어`로 실행

#### 리서처 트랙

Skill은 **분석 프레임워크/코드북**과 같습니다.
- 코드북: 인터뷰 분석 시 사용할 코드 체계를 미리 정의
- Skill: "인터뷰 분석", "설문 정리" 등을 미리 정의해두고 `/명령어`로 실행

#### PM 트랙

Skill은 **PRD 양식/기능 템플릿**과 같습니다.
- PRD 템플릿: 기능 개요, 유저 스토리, 성공 지표, 엣지 케이스 등 정해진 구조
- Skill: "PRD 작성", "스프린트 정리" 등을 미리 정의해두고 `/명령어`로 실행

#### Skill 구조

```
프로젝트/
└── .claude/
    └── skills/
        └── my-skill/
            └── SKILL.md    ← 이 파일에 레시피를 작성
```

SKILL.md에 작성한 내용은 `/my-skill` 명령어로 실행됩니다.

#### Skill 예시 (디자인 리뷰)

```markdown
# Design Review Skill

사용자가 Figma URL을 제공하면:
1. 파일 메타데이터 확인
2. 컴포넌트 사용 현황 정리
3. 디자인 시스템 정합성 체크 (컬러, 타이포, 간격)
4. 개선 제안 3가지 출력
```

#### Skill 예시 (인터뷰 분석)

```markdown
# Interview Analysis Skill

사용자가 인터뷰 녹취록을 제공하면:
1. 핵심 발견 5개 추출 (인용문 포함)
2. 감정 변화 흐름 정리
3. 숨겨진 니즈 도출
4. 다음 리서치 질문 제안
```

#### Skill 예시 (PRD 작성)

```markdown
# PRD Writer Skill

사용자가 기능 설명을 제공하면:
1. 기능 개요 작성
2. 유저 스토리 3개 도출
3. 성공 지표 (KPI) 제안
4. 엣지 케이스 3가지 정리
```

## EXECUTE

### 실습 1: CLAUDE.md 만들기

Claude Code 대화창에서 아래를 입력하세요:

```
내 claude-practice 폴더에 CLAUDE.md 파일을 만들어줘.
내용은 이렇게:
- 내 역할: {디자이너 또는 리서처 또는 PM}
- 선호 언어: 한국어
- 응답 스타일: 간결하게, 핵심만
- 자주 쓰는 도구: {Figma/Notion/Slack 중 해당하는 것}
```

### 실습 2: Skill 만들기

```
claude-practice 폴더에 간단한 Skill을 만들어줘.
.claude/skills/hello/SKILL.md 파일로,
내용은: "사용자가 /hello를 실행하면 오늘 날짜와 함께 격려 메시지를 출력한다"
```

### 실습 3: Skill 실행

새 Claude Code 세션을 claude-practice 폴더에서 시작하고:

```bash
# 터미널에서
cd ~/claude-practice
claude
```

그리고 대화창에서:
```
/hello
```

## QUIZ

### 퀴즈 1

```
질문: CLAUDE.md의 역할은?
옵션:
  A) 코드를 저장하는 파일
  B) Claude가 매 대화 시작 시 자동으로 읽는 영구 기억 파일
  C) Claude Code 설치 설정 파일
정답: B
해설: CLAUDE.md는 프로젝트 폴더에 두면 Claude Code가 자동으로 읽어
      매 대화에서 맥락을 유지합니다.
      디자이너에게는 '브랜드 가이드라인', 리서처에게는 '리서치 프로토콜',
      PM에게는 '제품 스펙 문서'와 같습니다.
```

### 퀴즈 2

```
질문: Skill 파일은 어디에 위치하나요?
옵션:
  A) 프로젝트 루트에 skill.txt
  B) .claude/skills/{이름}/SKILL.md
  C) ~/.claude/config.json
정답: B
해설: Skill은 .claude/skills/ 폴더 아래에 SKILL.md 파일로 작성합니다.
      폴더 이름이 곧 명령어가 됩니다 (예: hello 폴더 → /hello 명령어).
```
