# Block 2: Why CLI? - 왜 터미널인가 (~10분)

> 비개발자가 가장 궁금한 질문: "ChatGPT처럼 예쁜 화면이 있는데 왜 검은 화면을 쓰나요?"

---

## EXPLAIN

### "터미널 = 개발자 전용" 이라는 오해

터미널(CLI)은 텍스트로 명령을 내리는 도구입니다.
겁먹을 필요가 없는 이유:

1. **Claude Code에서 타이핑하는 건 프롬프트(자연어)입니다** — 코드가 아닙니다
2. **파일 탐색, 복사, 이동 같은 건 Claude가 대신 합니다** — 직접 명령어를 외울 필요 없습니다
3. **실수해도 Claude가 잡아줍니다** — "이 명령을 실행할까요?" 하고 물어봅니다

### GUI vs CLI: 왜 CLI가 더 강력한가?

| 비교 항목 | ChatGPT/웹 GUI | Claude Code (CLI) |
|----------|----------------|-------------------|
| 파일 접근 | 직접 업로드만 가능 | 내 컴퓨터 전체 파일 접근 |
| 외부 도구 | 제한적 플러그인 | MCP로 Figma/Notion/Slack 직접 연결 |
| 자동화 | 매번 복붙 | Skill로 반복 업무 레시피화 |
| 맥락 유지 | 대화 내 only | CLAUDE.md로 영구 기억 |
| 실행 | "이렇게 하세요" 안내 | 직접 실행까지 해줌 |

**핵심 차이**: GUI AI는 "답변"을 주고, CLI AI는 "실행"을 해줍니다.

### 비개발자를 위한 리프레이밍

**디자이너에게**: Figma에서도 Auto Layout, Variables 같은 "코드 같은" 기능을 쓰잖아요? Claude Code는 그것의 확장판입니다. Figma 플러그인 대신 "말로 하는 플러그인"이라고 생각하세요.

**리서처에게**: 설문 도구(Typeform, SurveyMonkey)도 결국 조건 분기, 필터링 같은 로직이죠? Claude Code는 그 로직을 "말로 설정하는 도구"입니다.

**PM에게**: Jira, Notion, Confluence 같은 도구에서 이슈 만들고, 상태 업데이트하고, 문서 정리하는 데 시간이 많이 들죠? Claude Code는 이 모든 도구를 "말 한마디로 조작하는 리모컨"입니다.

### 딱 알아야 할 터미널 지식 3가지

1. **`claude`** — Claude Code 시작
2. **`/mcp`** — 연결된 외부 도구 확인
3. **`/help`** — 도움말

이 3개만 알면 됩니다. 나머지는 Claude에게 자연어로 시키면 됩니다.

## EXECUTE

### 실습: CLI의 힘 직접 체험

Claude Code 대화창에서 아래를 순서대로 실행해보세요:

**1단계: 파일 생성**
```
"오늘의 할 일" 목록을 markdown 파일로 만들어줘.
항목은 3가지만: 아침 회의, 디자인 리뷰, 유저 테스트 정리
```

**2단계: 파일 수정**
```
방금 만든 파일에 "점심 약속" 항목을 추가하고, 각 항목에 예상 소요시간을 넣어줘
```

**3단계: 파일 검색**
```
내 claude-practice 폴더에 있는 모든 markdown 파일을 찾아줘
```

**체감 포인트**: 파일 경로나 명령어를 몰라도, 자연어로 모든 걸 할 수 있다는 것!

---

> 직접 실행해보세요! 3단계를 마치면 "완료"라고 입력해주세요.

## QUIZ

### 퀴즈 1

```
질문: Claude Code(CLI)가 ChatGPT(GUI)보다 유리한 점은?
옵션:
  A) 화면이 더 예쁘다
  B) 내 컴퓨터 파일에 직접 접근하고 실행까지 해준다
  C) 더 똑똑한 AI를 사용한다
정답: B
해설: CLI의 핵심 장점은 "직접 실행"입니다.
      파일 읽기/쓰기, 외부 도구 연결, 자동화가 GUI에서는 제한적이지만
      CLI에서는 자유롭게 가능합니다.
```

### 퀴즈 2

```
질문: Claude Code를 쓰기 위해 반드시 알아야 하는 터미널 명령어는 몇 개?
옵션:
  A) 약 20개
  B) 약 10개
  C) 3개면 충분 (claude, /mcp, /help)
정답: C
해설: Claude Code 자체가 자연어 인터페이스이기 때문에,
      터미널 명령어를 외울 필요가 거의 없습니다.
      시작(claude), 도구 확인(/mcp), 도움말(/help) 3개면 충분합니다.
```
